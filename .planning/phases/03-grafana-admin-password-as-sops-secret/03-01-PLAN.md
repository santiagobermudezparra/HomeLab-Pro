  <action>
1. Create `monitoring/configs/staging/kube-prometheus-stack/grafana-admin-secret.yaml` in plaintext using kubectl dry-run:

```bash
kubectl create secret generic grafana-admin-secret \
  --namespace monitoring \
  --from-literal=admin-user=admin \
  --from-literal=admin-password=watary \
  --dry-run=client -o yaml \
  > monitoring/configs/staging/kube-prometheus-stack/grafana-admin-secret.yaml
```

The file will contain a standard Kubernetes Secret with base64-encoded data values.

2. Encrypt the secret with SOPS using the project age key:

```bash
AGE_KEY=$(grep "age:" clusters/staging/.sops.yaml | awk '{print $NF}')

sops --age=${AGE_KEY} \
  --encrypt \
  --encrypted-regex '^(data|stringData)$' \
  --in-place monitoring/configs/staging/kube-prometheus-stack/grafana-admin-secret.yaml
```

3. Update `monitoring/configs/staging/kube-prometheus-stack/kustomization.yaml` to add the new secret as a resource. The final file must be:

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization
resources:
  - grafana-tls-secret.yaml
  - cloudnativepg-podmonitor.yaml
  - prometheus-rbac.yaml
  - grafana-admin-secret.yaml
```

(Preserve the existing three resources exactly — only append `grafana-admin-secret.yaml`.)
