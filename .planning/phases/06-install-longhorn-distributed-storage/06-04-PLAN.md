<objective>
Permanently demote `local-path` as the default StorageClass by adding `local-storage` to the `disable` list in K3s config on every cluster node, then restarting K3s. This is a node-level SSH operation — not a GitOps file change.

Purpose: K3s re-applies `storageclass.kubernetes.io/is-default-class: "true"` to `local-path` on every K3s server restart. A `kubectl patch` is not durable. The only permanent solution is `--disable=local-storage` in the K3s config. After this plan, `longhorn` is the sole default StorageClass.

IMPORTANT: This plan has no Git file changes. It is a node configuration change executed via SSH. It requires user verification of the cluster state before and after each node restart.

Output: K3s config updated on all nodes, K3s restarted, `local-path` StorageClass removed, `longhorn` confirmed as the only default.
</objective>

<task type="auto">
  <name>Task 2: Add local-storage to disable list on each node and restart K3s</name>
  <files></files>
  <action>
Edit K3s config on all nodes and restart K3s. Restart workers first, then control-plane last to minimize disruption.

**On each node, the change is:**
- If `disable:` list already exists: add `- local-storage` as a new list item
- If `disable:` list does not exist: create it with `local-storage` as the only entry
- If config.yaml does not exist: create it with just the disable block

**Step-by-step per node:**

Replace `<NODE_IP>` with the actual IPs from Task 1. Replace node names with actual names from `kubectl get nodes`.

```bash
# ── WORKER-01 ──
ssh <WORKER_01_IP> "
  # Show current config
  cat /etc/rancher/k3s/config.yaml 2>/dev/null
  
  # Add local-storage to disable list
  # If the file already has 'disable:' block, append to it
  # If not, create the file
  if grep -q 'disable:' /etc/rancher/k3s/config.yaml 2>/dev/null; then
    # Append to existing disable list
    sudo sed -i '/^disable:/a\  - local-storage' /etc/rancher/k3s/config.yaml
  else
    # Create or append to file
    echo -e 'disable:\n  - local-storage' | sudo tee -a /etc/rancher/k3s/config.yaml
  fi
  
  # Verify the edit
  cat /etc/rancher/k3s/config.yaml
"

# Restart K3s agent on worker-01
ssh <WORKER_01_IP> "sudo systemctl restart k3s-agent"

# Wait for worker-01 to rejoin (it will show Ready again)
kubectl wait node/<WORKER_01_NAME> --for=condition=Ready --timeout=120s
```

