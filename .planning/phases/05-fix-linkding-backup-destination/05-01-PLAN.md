<objective>
Wire the existing linkding ScheduledBackup to an S3-compatible object store by adding the
missing barmanObjectStore stanza to the linkding-postgres Cluster resource, creating an
encrypted S3 credentials secret, and activating backup-config.yaml in the kustomization.

Purpose: Satisfies BACK-02 — linkding backups currently exist as a ScheduledBackup CRD but
fail with "no barmanObjectStore section defined" because the Cluster has no backup destination
configured. This plan adds the destination and credentials so backups actually persist to
object storage.

Output:
- databases/staging/linkding/linkding-backup-s3-secret.yaml (new — SOPS-encrypted S3 credentials)
- databases/staging/linkding/postgresql-cluster.yaml (modified — add spec.backup.barmanObjectStore)
- databases/staging/linkding/kustomization.yaml (modified — uncomment backup-config.yaml, add secret)
</objective>
