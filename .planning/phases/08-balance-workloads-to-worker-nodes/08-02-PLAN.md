<objective>
Add `nodeAffinity` (soft preference) to all 8 single-replica app Deployments and to the Prometheus HelmRelease so that pods prefer to schedule on worker nodes rather than the control-plane.

Purpose: Satisfies SCHED-01 and SCHED-03 — worker nodes receive proportional workloads and control-plane is no longer forced to host the majority of app pods. This is a soft preference (preferredDuringSchedulingIgnoredDuringExecution) so pods can still land on control-plane if workers are unavailable or full.
Output: 8 updated deployment.yaml files and 1 updated HelmRelease values block.
</objective>

<interfaces>
<!-- How K3s labels nodes by role -->
<!-- Control-plane node has label: node-role.kubernetes.io/control-plane (exists)   -->
<!-- Worker nodes do NOT have this label (it is absent)                              -->
<!--                                                                                 -->
<!-- Strategy: use preferredDuringSchedulingIgnoredDuringExecution with              -->
<!-- DoesNotExist operator on node-role.kubernetes.io/control-plane                  -->
<!-- This means: prefer nodes where control-plane label DOES NOT EXIST = workers     -->
<!-- "Soft" preference: pods still schedule if all workers are full.                 -->

<!-- nodeAffinity stanza to add under spec.template.spec in each Deployment:         -->
```yaml
      affinity:
        nodeAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
            - weight: 100
              preference:
                matchExpressions:
                  - key: node-role.kubernetes.io/control-plane
                    operator: DoesNotExist
```

<!-- For Prometheus HelmRelease (monitoring/controllers/base/kube-prometheus-stack/release.yaml): -->
<!-- Add under values.prometheus.prometheusSpec:                                      -->
```yaml
        affinity:
          nodeAffinity:
            preferredDuringSchedulingIgnoredDuringExecution:
              - weight: 100
                preference:
                  matchExpressions:
                    - key: node-role.kubernetes.io/control-plane
                      operator: DoesNotExist
```

<!-- RULES:
  - NEVER use operator: In with values: [worker-01, worker-02] — those hardcode node names
  - NEVER use nodeName: worker-01 — that hardcodes a specific node
  - Use DoesNotExist on the control-plane ROLE LABEL — role-based, not name-based
  - weight: 100 = maximum preference (scheduler strongly prefers workers)
  - preferredDuring... = soft rule, pods still get scheduled if workers unavailable
-->
</interfaces>
