<task type="auto">
  <name>Task 2: Analyze root cause and determine if fix is needed</name>
  <files></files>
  <action>
Based on Task 1 logs, determine the root cause by following this decision tree:

**Pattern A: "external-host-error" or "ENOTFOUND" or DNS timeout**
  → Root Cause: Network/DNS issue reaching GitHub API, or transient network failure
  → Fix: None required if recent pods succeed (self-resolved). Document and monitor. If recurring: add RENOVATE_LOG_LEVEL=debug to ConfigMap to see details in future runs.

**Pattern B: "401 Unauthorized" or "403 Forbidden" or "Invalid authentication"**
  → Root Cause: RENOVATE_TOKEN is expired, revoked, or has insufficient permissions
  → Fix: User must provide a fresh GitHub token (user_setup task)

**Pattern C: "API rate limit exceeded" or "X-RateLimit-Remaining: 0"**
  → Root Cause: GitHub API rate limit hit
  → Fix: Could indicate too-frequent runs. Current schedule is @hourly. Decrease frequency or implement backoff. Confirm token has correct scopes (if a new token, might have lower limit).

**Pattern D: Container fails immediately or no logs**
  → Root Cause: Image pull failure, secret mount failure, or container crash on startup
  → Fix: Check `kubectl describe pod` for ImagePullBackOff, check secret exists, check renovate.json syntax

**Pattern E: Pods complete successfully (status=Succeeded) for both old and recent runs**
  → Root Cause: Error has self-resolved. Either transient network issue, or cluster recovered.
  → Fix: No fix needed. Pin the image tag to a stable version (best practice), update renovate.json to be more explicit.

**Pattern F: Cannot determine from logs (logs empty or truncated)**
  → Root Cause: Unknown
  → Fix: Add RENOVATE_LOG_LEVEL=debug to configmap for next run, wait 24h for another execution to gather debug logs, then revisit.
