<objective>
Migrate mealie's single PVC (mealie-data, 1Gi, 1.4MB data) from local-path to Longhorn.

Purpose: mealie is currently pinned to the control-plane node via local-path PV affinity. After migration to Longhorn, mealie can schedule on any node and its data is replicated across 2 nodes. After this plan, mealie will survive any single node failure.

Output: `apps/base/mealie/storage.yaml` updated with `storageClassName: longhorn`; mealie-data PVC bound on Longhorn; mealie pod Running with recipe data intact.
</objective>

<task type="auto">
  <name>Task 2: Live migration — backup mealie data, recreate PVC on Longhorn, restore data</name>
  <files></files>
  <read_first>
    - .planning/phases/07-migrate-pvcs-to-longhorn/07-RESEARCH.md (migration procedure, pitfall 3 about data corruption if app is running during copy)

  </read_first>
  <action>
Execute the following sequence imperatively against the live cluster. Do NOT add nodeSelector, nodeName, or nodeAffinity in the debug pod spec.

**Step 1 — Scale down mealie:**
```bash
kubectl scale deployment/mealie -n mealie --replicas=0
kubectl wait --for=delete pod -n mealie -l app=mealie --timeout=60s
```

**Step 2 — Spawn debug pod and copy data out (1.4MB):**
```bash
kubectl run debug-mealie \
  --image=busybox \
  --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"mealie-data"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/mealie-data"}]}]}}' \
  -n mealie
kubectl wait --for=condition=Ready pod/debug-mealie -n mealie --timeout=60s
mkdir -p /tmp/mealie-backup
kubectl cp mealie/debug-mealie:/mealie-data/. /tmp/mealie-backup/
kubectl delete pod/debug-mealie -n mealie --grace-period=0
```

**Step 3 — Delete old PVC and apply updated storage.yaml:**
```bash
kubectl delete pvc mealie-data -n mealie
kubectl apply -f apps/base/mealie/storage.yaml
kubectl wait pvc/mealie-data -n mealie \
  --for=jsonpath='{.status.phase}'=Bound --timeout=120s
```

**Step 4 — Restore data via new debug pod:**
```bash
kubectl run debug-mealie \
  --image=busybox \
  --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"mealie-data"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/mealie-data"}]}]}}' \
  -n mealie
kubectl wait --for=condition=Ready pod/debug-mealie -n mealie --timeout=60s
kubectl cp /tmp/mealie-backup/. mealie/debug-mealie:/mealie-data/
kubectl delete pod/debug-mealie -n mealie --grace-period=0
```

**Step 5 — Scale up and verify:**
```bash
kubectl scale deployment/mealie -n mealie --replicas=1
kubectl wait --for=condition=Ready pod -n mealie -l app=mealie --timeout=120s
```

**Cleanup:** `rm -rf /tmp/mealie-backup`
