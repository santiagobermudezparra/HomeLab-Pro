<objective>
Migrate all 7 audiobookshelf PVCs from local-path to Longhorn in one atomic operation.

Purpose: audiobookshelf mounts 7 PVCs simultaneously (/config, /metadata, /audiobooks, /podcasts, /ebooks, /comics, /videos). All 7 must be migrated while the app is scaled to 0 — migrating them one-at-a-time with the app running between migrations would cause mount failures. After this plan, audiobookshelf will survive any single node failure, and its data volumes are decoupled from the control-plane node.

Output: `apps/base/audiobookshelf/storage.yaml` updated with `storageClassName: longhorn` on all 7 PVCs; all 7 PVCs bound on Longhorn; audiobookshelf pod Running with config and metadata intact.
</objective>

Execute the following sequence imperatively. The app stays at 0 replicas for the ENTIRE migration — do NOT scale up until ALL 7 PVCs are migrated and data is restored. Do NOT add nodeSelector, nodeName, or nodeAffinity to debug pod specs.

**Step 1 — Scale down audiobookshelf:**
```bash
kubectl scale deployment/audiobookshelf -n audiobookshelf --replicas=0
kubectl wait --for=delete pod -n audiobookshelf -l app=audiobookshelf --timeout=60s
```

**Step 2 — Back up audiobookshelf-config (320KB — has actual data):**
```bash
kubectl run debug-abs-config \
  --image=busybox --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"audiobookshelf-config"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/abs-data"}]}]}}' \
  -n audiobookshelf
kubectl wait --for=condition=Ready pod/debug-abs-config -n audiobookshelf --timeout=60s
mkdir -p /tmp/abs-config-backup
kubectl cp audiobookshelf/debug-abs-config:/abs-data/. /tmp/abs-config-backup/
kubectl delete pod/debug-abs-config -n audiobookshelf --grace-period=0
```

**Step 3 — Back up audiobookshelf-metadata (80KB — has actual data):**
```bash
kubectl run debug-abs-meta \
  --image=busybox --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"audiobookshelf-metadata"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/abs-data"}]}]}}' \
  -n audiobookshelf
kubectl wait --for=condition=Ready pod/debug-abs-meta -n audiobookshelf --timeout=60s
mkdir -p /tmp/abs-metadata-backup
kubectl cp audiobookshelf/debug-abs-meta:/abs-data/. /tmp/abs-metadata-backup/
kubectl delete pod/debug-abs-meta -n audiobookshelf --grace-period=0
```

**Step 4 — Delete all 7 old PVCs and apply updated storage.yaml:**
```bash
kubectl delete pvc audiobookshelf-config audiobookshelf-metadata audiobookshelf-audiobooks \
  audiobookshelf-podcasts audiobookshelf-ebooks audiobookshelf-comics audiobookshelf-videos \
  -n audiobookshelf
kubectl apply -f apps/base/audiobookshelf/storage.yaml
for PVC in audiobookshelf-config audiobookshelf-metadata audiobookshelf-audiobooks audiobookshelf-podcasts audiobookshelf-ebooks audiobookshelf-comics audiobookshelf-videos; do
  kubectl wait pvc/$PVC -n audiobookshelf \
    --for=jsonpath='{.status.phase}'=Bound --timeout=120s
  echo "$PVC: Bound"
done
```

**Step 5 — Restore audiobookshelf-config:**
```bash
kubectl run debug-abs-config \
  --image=busybox --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"audiobookshelf-config"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/abs-data"}]}]}}' \
  -n audiobookshelf
kubectl wait --for=condition=Ready pod/debug-abs-config -n audiobookshelf --timeout=60s
kubectl cp /tmp/abs-config-backup/. audiobookshelf/debug-abs-config:/abs-data/
kubectl delete pod/debug-abs-config -n audiobookshelf --grace-period=0
```

**Step 6 — Restore audiobookshelf-metadata:**
```bash
kubectl run debug-abs-meta \
  --image=busybox --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"audiobookshelf-metadata"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/abs-data"}]}]}}' \
  -n audiobookshelf
kubectl wait --for=condition=Ready pod/debug-abs-meta -n audiobookshelf --timeout=60s
kubectl cp /tmp/abs-metadata-backup/. audiobookshelf/debug-abs-meta:/abs-data/
kubectl delete pod/debug-abs-meta -n audiobookshelf --grace-period=0
```

**Note on empty PVCs (audiobooks, podcasts, ebooks, comics, videos):** These contain only ~4KB (lost+found directory). No backup needed — the new empty Longhorn volumes are equivalent. The delete and recreate in Step 4 is sufficient.

**Step 7 — Scale up and verify:**
```bash

kubectl wait --for=condition=Ready pod -n audiobookshelf -l app=audiobookshelf --timeout=120s
```
