<task type="auto">
  <name>Task 2: Live migration — backup both filebrowser PVCs, recreate on Longhorn, restore data</name>
  <files></files>
  <read_first>
    - .planning/phases/07-migrate-pvcs-to-longhorn/07-RESEARCH.md (migration procedure, pitfall 3 about copying while app is running)
    - apps/base/filebrowser/storage.yaml (confirm storageClassName: longhorn is present on both PVCs)
  </read_first>
  <action>
Execute the following sequence imperatively against the live cluster. Both PVCs are migrated while the app is scaled to 0 — do NOT scale up between PVC migrations. Do NOT add nodeSelector, nodeName, or nodeAffinity in debug pod specs.

**Step 1 — Scale down filebrowser:**
```bash
kubectl scale deployment/filebrowser -n filebrowser --replicas=0
kubectl wait --for=delete pod -n filebrowser -l app=filebrowser --timeout=60s
```

**Step 2 — Copy out filebrowser-db (44KB SQLite database):**
```bash
kubectl run debug-fb-db \
  --image=busybox \
  --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"filebrowser-db"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/fb-db"}]}]}}' \
  -n filebrowser
kubectl wait --for=condition=Ready pod/debug-fb-db -n filebrowser --timeout=60s
mkdir -p /tmp/filebrowser-db-backup
kubectl cp filebrowser/debug-fb-db:/fb-db/. /tmp/filebrowser-db-backup/
kubectl delete pod/debug-fb-db -n filebrowser --grace-period=0
```

**Step 3 — Copy out filebrowser-files (~4KB, nearly empty):**
```bash
kubectl run debug-fb-files \
  --image=busybox \
  --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"filebrowser-files"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/fb-files"}]}]}}' \
  -n filebrowser
kubectl wait --for=condition=Ready pod/debug-fb-files -n filebrowser --timeout=60s
mkdir -p /tmp/filebrowser-files-backup
kubectl cp filebrowser/debug-fb-files:/fb-files/. /tmp/filebrowser-files-backup/
kubectl delete pod/debug-fb-files -n filebrowser --grace-period=0
```

**Step 4 — Delete old PVCs and apply updated storage.yaml:**
```bash
kubectl delete pvc filebrowser-db -n filebrowser
kubectl delete pvc filebrowser-files -n filebrowser
kubectl apply -f apps/base/filebrowser/storage.yaml
kubectl wait pvc/filebrowser-db -n filebrowser \
  --for=jsonpath='{.status.phase}'=Bound --timeout=120s
kubectl wait pvc/filebrowser-files -n filebrowser \
  --for=jsonpath='{.status.phase}'=Bound --timeout=120s
```

**Step 5 — Restore filebrowser-db:**
```bash
kubectl run debug-fb-db \
  --image=busybox \
  --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"filebrowser-db"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/fb-db"}]}]}}' \
  -n filebrowser
kubectl wait --for=condition=Ready pod/debug-fb-db -n filebrowser --timeout=60s
kubectl cp /tmp/filebrowser-db-backup/. filebrowser/debug-fb-db:/fb-db/
kubectl delete pod/debug-fb-db -n filebrowser --grace-period=0
```

**Step 6 — Restore filebrowser-files:**
```bash
kubectl run debug-fb-files \
  --image=busybox \
  --restart=Never \
  --overrides='{"spec":{"volumes":[{"name":"data","persistentVolumeClaim":{"claimName":"filebrowser-files"}}],"containers":[{"name":"debug","image":"busybox","command":["sleep","600"],"volumeMounts":[{"name":"data","mountPath":"/fb-files"}]}]}}' \
  -n filebrowser
kubectl wait --for=condition=Ready pod/debug-fb-files -n filebrowser --timeout=60s
kubectl cp /tmp/filebrowser-files-backup/. filebrowser/debug-fb-files:/fb-files/
kubectl delete pod/debug-fb-files -n filebrowser --grace-period=0
```

**Step 7 — Scale up and verify:**
```bash
kubectl scale deployment/filebrowser -n filebrowser --replicas=1
kubectl wait --for=condition=Ready pod -n filebrowser -l app=filebrowser --timeout=120s
```

**Cleanup:** `rm -rf /tmp/filebrowser-db-backup /tmp/filebrowser-files-backup`
