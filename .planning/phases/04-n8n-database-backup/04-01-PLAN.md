**Step 1 ŌĆö Create databases/staging/n8n/backup-config.yaml**

Create the file with the following exact content (mirrors linkding/backup-config.yaml with n8n-specific values):

```yaml
apiVersion: postgresql.cnpg.io/v1
kind: ScheduledBackup
metadata:
  name: n8n-backup
  namespace: n8n
spec:
  # Schedule: Daily at 3 AM
  schedule: "0 3 * * *"

  # Backup immediately on creation
  immediate: true

  # Reference to the cluster
  cluster:
    name: n8n-postgresql-cluster

  # Retention policy
  backupOwnerReference: cluster
```

Key values (do not deviate):
- `metadata.name`: `n8n-backup` (not `n8n-postgresql-cluster-backup`, not `linkding-backup`)
- `metadata.namespace`: `n8n`
- `spec.schedule`: `"0 3 * * *"` (5-field standard cron ŌĆö daily at 03:00 UTC; NOT 6-field Go cron format)
- `spec.immediate`: `true` (triggers a backup on creation)
- `spec.cluster.name`: `n8n-postgresql-cluster` (exact cluster name from postgresql-cluster.yaml)
- `spec.backupOwnerReference`: `cluster`
- Do NOT add `barmanObjectStore` or `destinationPath` ŌĆö object storage is Phase 5 scope
